# Project Orchestrator

1. Identify the owning backend module and mirrored frontend feature.
2. Update `packages/shared` before duplicating contracts.
3. Keep persistence in repositories and business workflows in services.
4. Update setup and project guidance when assumptions change.
5. Run focused checks, then database validation, tests, type checking, and builds.
6. Keep the starter small; add infrastructure only when a concrete student feature needs it.
