# Naming Convention

- Folders and route segments: lowercase kebab-case.
- React components and exported types: PascalCase.
- Functions and variables: camelCase.
- Environment constants: UPPER_SNAKE_CASE.
- API routes: plural nouns where natural.
- Name files and symbols by domain intent rather than implementation detail.
- Prefer clear beginner-readable names over abbreviations.
